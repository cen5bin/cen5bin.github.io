IDEA远程调试代码

发表于 2016-03-28   |   分类于 效率   |     |   阅读次数

作为一个经常在服务端上做开发的人，难免需要调试服务器上的代码，以前采用的办法是，在代码中加入大量的log，然后通过查看日志文件来寻找问题，后来知道了其实远程登录的情况下也是可以打开图形界面的，于是在服务器上装了相关的IDE，然后进行单步调试，但是这样感觉会很慢，再后来忍无可忍，就寻找了新的办法，发现IDEA可以直接进行远程调试，当时感觉很神奇……

JVM参数设置

只有在启动java程序时设定相应的参数，才可以支持远程调试，参数如下，address后面的表示端口号，这里我设置了9999，除了这个端口号，其他都不用改

IDEA设置

  1. 先在工具栏上找到一个下拉菜单，然后选择“Edit Configurations”，或者在“Run”菜单下面也可以找到
  2. 增加一个配置，选择remote
  3. 设置相关的参数，包括服务器的ip，以及debug的端口号，端口号要和jvm参数中的对应
  4. 点击OK即可

开始调试

如下图所示，之前设置的configuration已经生效了


在服务器上启动需要调试的程序（带上jvm参数），然后在本地的IDEA中点击上图中的虫子，即可
具体的调试，如何加断点，单步，查看变量值就不介绍了
